## INSTRUÇÕES PARA PREENCHIMENTO E OTIMIZAÇÃO

### DIRETRIZES GERAIS DE PREENCHIMENTO

1. **TÍTULO DO DOCUMENTO**
   - Use um título específico e descritivo que contenha os conceitos-chave
   - Incorpore termos técnicos relevantes para busca vetorial
   - Evite títulos genéricos ou ambíguos

2. **METADADOS SEMÂNTICOS (JSON)**
   - **doc_id**: Crie um ID estruturado e significativo (ex: DOC-ML-027-2023)
   - **primary_concepts**: Liste 3-7 conceitos fundamentais ordenados por importância
   - **question_embeddings**: Crie 7-10 perguntas naturais que o documento responde diretamente
   - **complexity_level**: Avalie honestamente o nível técnico (básico/intermediário/avançado/especialista)
   - **key_concepts**: Identifique conceitos críticos diferenciadores deste documento

3. **SUMÁRIO EXECUTIVO**
   - Concentre a maior densidade semântica possível em 250-350 palavras
   - Inclua TODOS os conceitos-chave e terminologia especializada
   - Estruture como uma resposta autocontida sobre o tema principal
   - Evite linguagem promocional, vaga ou excessivamente simplificada

4. **MAPA CONCEITUAL**
   - Crie uma representação gráfica textual das relações entre conceitos
   - Explicite hierarquias, influências e contrastes entre conceitos
   - Use terminologia técnica precisa nas definições

### DIRETRIZES DE ESTRUTURAÇÃO DE CONTEÚDO

5. **SEÇÕES PRINCIPAIS**
   - Inicie cada seção com um resumo denso em conceitos (1-2 frases)
   - Mantenha consistência na hierarquia de títulos (H1 > H2 > H3)
   - Use numeração para estabelecer sequência lógica clara

6. **CHUNKING COGNITIVO**
   - Cada subseção deve representar uma unidade conceitual coesa
   - Mantenha chunking baseado em conceitos completos (não tamanho fixo)
   - Estruture parágrafos com sentence lead (frase tópico inicial forte)

7. **ELEMENTOS ESPECIAIS**
   - Use tabelas markdown para informações comparativas e estruturadas
   - Crie diagramas textuais para representar relações complexas
   - Utilize listas numeradas para processos sequenciais
   - Empregue listas com marcadores para características não-ordenadas

### DIRETRIZES DE OTIMIZAÇÃO SEMÂNTICA

8. **ENRIQUECIMENTO TERMINOLÓGICO**
   - Defina formalmente termos técnicos na primeira ocorrência
   - Use terminologia consistente ao longo do documento (evite sinônimos para conceitos críticos)
   - Inclua variações relevantes de termos-chave para melhorar recuperação

9. **DENSIDADE INFORMACIONAL**
   - Maximize a relação sinal/ruído evitando linguagem redundante
   - Inclua dados quantitativos específicos (números, parâmetros, limites)
   - Concentre exemplos técnicos concretos em vez de explicações genéricas

10. **CONTEXTO REFERENCIAL**
    - Vincule conceitos a padrões da indústria ou publicações acadêmicas
    - Estabeleça comparações explícitas com abordagens alternativas
    - Inclua referências cruzadas a outros documentos no mesmo domínio

### INSTRUÇÕES FINAIS DE VALIDAÇÃO

11. **VERIFICAÇÕES PRÉ-PUBLICAÇÃO**
    - Confirme que cada seção pode funcionar como unidade de conhecimento independente
    - Verifique se termos técnicos estão definidos antes de serem utilizados extensivamente
    - Garanta que perguntas em question_embeddings são respondidas explicitamente no texto
    - Teste a navegabilidade cognitiva: a estrutura hierárquica conduz logicamente o leitor
    - Avalie a densidade semântica: cada parágrafo contém múltiplos conceitos relevantes
    - Confirme precisão técnica: valores, parâmetros e relações estão corretos e atualizados

## RECOMENDAÇÕES AVANÇADAS DE OTIMIZAÇÃO RAG

### ENGENHARIA DE DENSIDADE SEMÂNTICA

1. **CONTROLANDO PROXIMIDADE VETORIAL**
   - Agrupe termos relacionados em parágrafos específicos (cria "ilhas semânticas")
   - Separe claramente conceitos distintos em seções separadas para evitar interferência de embedding
   - Use repetição estratégica de termos-chave com diferentes contextos explanatórios

2. **OTIMIZAÇÃO DE RECUPERAÇÃO MULTIFACETADA**
   - Crie múltiplos pontos de ancoragem semântica para conceitos importantes
   - Varie a formulação de definições-chave (técnica, aplicada, comparativa)
   - Desenvolva exemplos que ilustrem diferentes aspectos do mesmo conceito

3. **ESTRATÉGIAS DE SOBREPOSIÇÃO CONTROLADA**
   - Implemente sobreposição intencional de 10-15% entre seções relacionadas
   - Use frases de transição que incorporem termos-chave da seção anterior e seguinte
   - Estabeleça pontes conceituais explícitas entre seções relacionadas não consecutivas

### CONSIDERAÇÕES AVANÇADAS DE APLICAÇÃO

Este template representa um formato otimizado para documentação técnica em sistemas RAG. A estrutura proposta:

1. **MAXIMIZA PRECISÃO DE RECUPERAÇÃO**

   - Via chunking semanticamente coerente

2. **FACILITA GERAÇÃO CONTEXTUAL**

3. **SUPORTA RACIOCÍNIO AVANÇADO**
